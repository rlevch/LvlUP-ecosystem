# Получение API-ключей ЮKassa для LevelUp

> Инструкция для подключения платёжной системы ЮKassa к платформе SCORE Coaching / LevelUp.
> Дата: 25.03.2026

---

## 1. Регистрация в ЮKassa

1. Перейдите на сайт [yookassa.ru](https://yookassa.ru) и нажмите «Подключить».
2. Заполните заявку: укажите данные организации (ИП или юрлицо), сайт и контактную информацию.
3. Загрузите необходимые документы (скан паспорта для ИП, устав для ООО, и т.д.).
4. Дождитесь проверки — обычно занимает 1–3 рабочих дня. После одобрения вы получите доступ в личный кабинет.

---

## 2. Получение shopId

**shopId** — это идентификатор вашего магазина в системе ЮKassa.

1. Войдите в [личный кабинет ЮKassa](https://yookassa.ru/my).
2. В верхней панели отображается название вашего магазина — **shopId** указан рядом с ним.
3. Также shopId можно найти в разделе **Настройки → Данные магазина**.
4. Скопируйте значение и сохраните — оно понадобится для конфигурации.

---

## 3. Выпуск секретного ключа (Secret Key)

**Секретный ключ** используется для авторизации API-запросов вместе с shopId.

1. В личном кабинете перейдите в раздел **Интеграция → Ключи API**.
2. Нажмите кнопку **«Выпустить ключ»**.
3. Подтвердите действие — на привязанный номер телефона придёт SMS с кодом.
4. Введите код подтверждения.
5. **Сразу скопируйте и сохраните ключ** — ЮKassa не хранит секретный ключ после показа. Если потеряете, придётся выпускать новый.

**Формат:** `live_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`

---

## 4. Тестовый режим (Sandbox)

Перед подключением к боевым платежам рекомендуется проверить интеграцию в тестовом режиме.

### Тестовые данные

| Параметр | Значение |
|----------|----------|
| API URL | `https://api.yookassa.ru/v3/` |
| Тестовый shopId | Тот же shopId из личного кабинета |
| Тестовый ключ | Выпускается отдельно в разделе **Интеграция → Ключи API** (переключатель «Тестовый магазин») |

### Тестовые карты

| Номер карты | Сценарий |
|-------------|----------|
| `5555 5555 5555 4444` | Успешный платёж |
| `5555 5555 5555 4445` | Отказ: недостаточно средств |
| `5555 5555 5555 4446` | Отказ: отклонено банком |

Срок действия — любой будущий, CVC — любые 3 цифры.

## 5. Настройка вебхуков (Webhooks)

Вебхуки нужны, чтобы платформа получала уведомления о статусах платежей.

1. В личном кабинете перейдите в **Интеграция → HTTP-уведомления**.
2. Укажите URL для уведомлений:
   - Для продакшена: `https://api.levelup-academy.ru/webhooks/yookassa`
   - Для тестового окружения: `https://api-staging.levelup-academy.ru/webhooks/yookassa`
3. Выберите события:
   - `payment.succeeded` — платёж прошёл
   - `payment.canceled` — платёж отменён
   - `refund.succeeded` — возврат выполнен
   - `payment.waiting_for_capture` — платёж ожидает подтверждения (для двухстадийных платежей)

## 6. Где использовать ключи в проекте

Ключи хранятся в переменных окружения. **Никогда не коммитьте ключи в репозиторий.**

### 6.1. Подключение на VPS #1 (score-app-01, 111.88.113.107)

API-gateway работает через PM2. Ключи задаются в файле `ecosystem.config.cjs`.

**Сервер:** VPS #1 · **Пользователь:** `deploy`

```bash
nano /home/deploy/levelup-monorepo/ecosystem.config.cjs
```

Найдите секцию `env` внутри `api-gateway` и добавьте три переменные:

```js
module.exports = {
  apps: [
    {
      name: 'api-gateway',
      // ... остальные параметры ...
      env: {
        // ... существующие переменные ...
        YOOKASSA_SHOP_ID: '123456',                            // ← ваш shopId
        YOOKASSA_SECRET_KEY: 'live_XXXXXXXXXXXXXXXXXXXXXXXX',  // ← ваш секретный ключ
        YOOKASSA_RETURN_URL: 'https://levelup-academy.ru/dashboard/finances',
        PLATFORM_COMMISSION_PCT: '15',                         // комиссия платформы, %
      },
    },
    // ...
  ],
};
```

После редактирования перезапустите PM2:

```bash
cd /home/deploy/levelup-monorepo
pm2 restart api-gateway
pm2 logs api-gateway --lines 10
```

### 6.2. Как работает dev-fallback (текущий режим)

Сейчас платформа работает **без реальных ключей ЮKassa**. При создании платежа API проверяет наличие `YOOKASSA_SHOP_ID`:

- **Ключи НЕ заданы** (текущее состояние) — платёж автоматически помечается как `succeeded`, деньги не списываются. Это позволяет тестировать весь flow без реального эквайринга.
- **Ключи заданы** — платёж создаётся через API ЮKassa, клиент перенаправляется на страницу оплаты, далее работают вебхуки.

### 6.3. Переключение на боевой режим

1. Получите shopId и секретный ключ (разделы 2–3 выше).
2. Настройте вебхуки в ЛК ЮKassa (раздел 5 выше). URL вебхука:
   ```
   https://api.levelup-academy.ru/api/payments/webhook
   ```
3. Пропишите ключи в `ecosystem.config.cjs` (раздел 6.1).
4. Перезапустите PM2: `pm2 restart api-gateway`.
5. Сделайте тестовый платёж с тестовой картой (раздел 4).
6. Убедитесь, что статус платежа обновился в базе.
7. Переключите ключи с тестовых на боевые.

### 6.4. Формат авторизации

ЮKassa использует HTTP Basic Auth:

```
Authorization: Basic base64(shopId:secretKey)
```

В коде API (services/api/src/routes/payments.ts) авторизация формируется автоматически из переменных окружения.

## 7. Split Payments (для школ на платформе)

Для функциональности расщепления платежей (школа получает доход, платформа — комиссию) каждая школа должна быть подключена как субаккаунт ЮKassa.

1. Школа регистрируется в ЮKassa через партнёрскую ссылку платформы.
2. После верификации школа получает свой shopId.
3. Платформа добавляет shopId школы в настройки tenant-а (`/admin/settings/payments`).
4. При оплате курса платёж расщепляется автоматически: основная часть → школе, комиссия (3–10% в зависимости от тарифа) → платформе.

> На этапе MVP Split Payments не подключаются — все платежи идут через центральный аккаунт платформы. Split вводится на Фазе 2.
